结论：**当前方案可行**，且与现有 C++ 封装和前端用法一致。下面是依据和需要留意的点。

### 1. API 对应关系

- 导出：代码里是 **`exportBREP`**（不是 exportToBrep），签名是 `exportBREP(shapes: TopoDS_Shape[]): Uint8Array`。对单个 `ShapeNode` 的 shape 导出时，用 `exportBREP([node.shape])` 即可（单元素数组）。
- 导入：**`importBREP(buffer: Uint8Array): TopoDS_Shape`** 已存在。`examples/src/cases/exchange/index.ts` 里 BREP 导入后已通过「捏造」ShapeNode（`shape` + `getChildren: () => []`）参与渲染，说明「前端捏造 ShapeNode」这条路径是通的。

### 2. 方案与现有实现的对齐

- **颗粒度以 ShapeNode 为单位**：有 shape 的节点用 `exportBREP([node.shape])` 得到一段 BRep 二进制；Compound/CompoundSolid 只负责树结构（通过 `getChildren`），不再把整棵树压成一块 BRep（除非你额外做「整树导出为一文件」的兼容）。
- **Solid 不解析**：即该节点若是完整 Solid，就整块导出为一段 BRep，不拆子 shape，与现有逻辑一致。
- **单独点/线/面**：可作为独立 ShapeNode，各自一段 BRep，没有问题。
- **Compound / CompoundSolid**：只存树结构并递归 `getChildren`，子节点各自带自己的 shape 的 BRep，与 `shapeNodeToBrepRenderNode`、`getCompoundChildren` 的递归方式一致。

现有 `collectShapesFromShapeNode` 已支持从整棵 ShapeNode 树收集所有 shape，若将来需要「整树导出为单一大 BRep 文件」也可以复用。

### 3. 自定义文件格式建议

- **载体**：用 JSON 做外层格式，便于编辑和扩展。树结构存 `name`、`color`、`children` 顺序（或 id 引用）；每个节点的 BRep 存为 **base64 字符串**（或二进制块 + 长度/偏移，看体积需求）。
- **导出流程**：遍历 ShapeNode 树；对每个有 `shape` 的节点调用 `Exchange.exportBREP([node.shape])`，得到 `Uint8Array`，再 base64 写入该节点在 JSON 中的字段；无 shape 仅有 children 的节点只写树结构。
- **导入流程**：解析 JSON，按树结构递归；对每个带 BRep 的节点 `Exchange.importBREP(uint8Array)` 得到 `TopoDS_Shape`，再拼成 ShapeNode（`shape` + `name`/`color` + `getChildren` 返回下一层顺序）。

### 4. 一个小细节（可选）

`exportBREP([singleShape])` 内部会用 `Compound::fromShapes([singleShape])` 再 `BRepTools::Write`，所以写出的 BRep 是「包含一个子 shape 的 Compound」。`importBREP` 读回的是这个 **Compound**，不是裸的 Solid/Shell 等。若你希望还原后 `node.shape` 仍是原来的类型（例如 Solid），导入后可以判断：若 `shape.ShapeType() === Compound` 且 `shape.children().length === 1`，则取 `shape.children()[0]` 作为该节点的 `shape`；否则直接用读回的 Compound 也可以。

已在 SDK 的 `examples/src/sdk/exchange.ts` 中完成接口定义与初步实现，具体如下。

**1. 类型（已 export，供调用方使用）**

- `SerializedShapeNode`：单节点 JSON 结构，含可选 `name`、`color`、`brep`（base64）、`children`。
- `SerializedShapeDocument`：根结构 `{ version: 1, root: SerializedShapeNode }`，便于后续格式版本演进。

**2. Exchange 新增静态方法**

- `Exchange.serializeToCustomFormat(shapeNode: ShapeNode): string`  
  将整棵 ShapeNode 树序列化为上述自定义格式的 JSON 字符串（有 shape 的节点会调用 `exportBREP([node.shape])` 并 base64 写入）。

- `Exchange.deserializeFromCustomFormat(json: string): ShapeNode`  
  从 JSON 反序列化回 ShapeNode 树（内部对每个 `brep` 调用 `importBREP`，并做「单子 Compound 解包」），返回的树可直接交给 `shapeNodeToBrepRenderNode` 等使用。

**3. 使用示例**

```ts
import { Exchange } from "@/sdk/exchange";
import { shapeNodeToBrepRenderNode } from "@/common/shape-converter";

// 导出
const json = Exchange.serializeToCustomFormat(globalShapeNode);

// 导入
const shapeNode = Exchange.deserializeFromCustomFormat(json);
const sceneObj = shapeNodeToBrepRenderNode(shapeNode, defaultMaterial);
```
